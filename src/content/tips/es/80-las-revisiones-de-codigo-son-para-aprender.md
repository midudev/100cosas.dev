Sarah Drasner, VP de Developer Experience en Netlify, promueve una cultura de code review positiva: **el objetivo es mejorar el código y el equipo, no demostrar quién sabe más**.

## Reviews que ayudan

```markdown
❌ "Este código es malo"
✅ "¿Qué te parece si extraemos esta lógica a una función? Sería más fácil de testear"

❌ "No uses var"
✅ "Prefiero const aquí porque el valor no cambia. ¿Qué opinas?"

❌ "Esto no va a funcionar"
✅ "Me preocupa que esto falle cuando X. ¿Has considerado ese caso?"
```

## El checklist de una buena review

1. **¿El código hace lo que debe?**
2. **¿Es fácil de entender?**
3. **¿Hay tests?**
4. **¿Puedo aprender algo de este código?**

## Reflexión final

Las mejores code reviews son conversaciones, no juicios. El autor aprende, el revisor aprende, y el código mejora.
