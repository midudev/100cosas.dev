Barbara Liskov, ganadora del Turing Award, enfatiza entender el problema: **la solución correcta al problema equivocado sigue siendo incorrecta**.

## El error común

```markdown
Cliente: "Necesito exportar a Excel"
Dev: *construye exportación a Excel*

Lo que el cliente necesitaba: ver datos en una tabla
Lo que hubiera funcionado: tabla HTML con filtros
```

## Las preguntas correctas

```markdown
- ¿Qué problema estás tratando de resolver?
- ¿Cómo lo haces actualmente?
- ¿Qué pasaría si no hacemos esto?
- ¿Cómo sabremos que funcionó?
```

## Reflexión final

La mitad de tu trabajo es entender qué construir. La otra mitad es construirlo.
