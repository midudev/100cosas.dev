Kent C. Dodds ha luchado contra el dogmatismo: **lo que funciona para Google no necesariamente funciona para tu startup de 3 personas**.

## "Mejores prácticas" que dependen

```markdown
Microservicios: Genial para Netflix, overkill para tu MVP
TDD estricto: Útil para librerías, excesivo para prototipos
TypeScript: Esencial para equipos grandes, opcional para scripts pequeños
```

## La pregunta correcta

No preguntes "¿Cuál es la mejor práctica?"

Pregunta "¿Cuál es la mejor práctica **para mi contexto**?"

## Reflexión final

Los dogmas son cómodos porque evitan pensar. Pero el buen engineering es exactamente eso: pensar en el contexto específico.
