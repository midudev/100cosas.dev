Wes Bos has taught web development to hundreds of thousands of people with a simple principle: **stop reading infinite documentation and start writing code**.

## The infinite tutorial trap

```markdown
Monday: "I'm going to learn React"
- Read the complete documentation
- Watch 3 videos of "React in 2024"
- Compare Next.js vs Remix vs Astro

Friday: 
- Still hasn't written a line of code
- Feels "not ready yet"
- Finds another "better" tutorial
```

This is called **Tutorial Hell** and it's a form of procrastination disguised as productivity.

## Wes's method

### JavaScript30

Wes's free JavaScript30 course has a simple premise:

```markdown
30 days → 30 projects → No frameworks

Day 1: Drum Kit
- You don't read about events
- You MAKE a drum kit with events

Day 2: Clock
- You don't study Date()
- You MAKE a working clock
```

You learn by doing, not by preparing to do.

## Why it works better

### 1. Muscle memory

```javascript
// After writing this 50 times:
document.querySelector('.element').addEventListener('click', () => {
  // ...
});

// You no longer need to think about it. It's automatic.
```

### 2. Real context

Reading about `async/await` in documentation is abstract. Using it to fetch from a real API gives you context.

### 3. Errors as teachers

The errors you find by doing teach you more than any tutorial.

## How to apply it

### The 20/80 rule

```markdown
20% reading → Understand basic concepts
80% doing → Apply, fail, fix, repeat
```

### Minimal projects

You don't need original ideas. Clone existing things:

```markdown
Learning React: Make a Twitter clone (just the feed)
Learning Node: Make a REST API for TODOs
Learning CSS: Replicate Stripe's homepage
```

## Final reflection

Wes says: "The best time to start a project is now, not when you know more". You'll never know "enough". Knowledge comes from doing, not from preparing to do.
