Sarah Drasner, VP of Developer Experience at Netlify, promotes a positive code review culture: **the goal is to improve code and the team, not prove who knows more**.

## Reviews that help

```markdown
❌ "This code is bad"
✅ "What do you think about extracting this logic to a function? It would be easier to test"

❌ "Don't use var"
✅ "I prefer const here because the value doesn't change. What do you think?"
```

## Final reflection

The best code reviews are conversations, not judgments. The author learns, the reviewer learns, and the code improves.
